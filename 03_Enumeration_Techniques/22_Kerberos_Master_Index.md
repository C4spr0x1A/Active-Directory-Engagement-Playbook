## üõ†Ô∏è **COMPREHENSIVE ENUMERATION TOOLS**

### **üîß Microsoft‚ÄëSigned Tools**
- **ActiveDirectory Module**: Signed discovery with minimal footprint
  - `Get-ADDomain` ‚Äî Domain basics (KDCs, PDC)
  - `Get-ADForest` ‚Äî Forest trusts and domains
  - `Get-ADUser -Filter * -Properties ServicePrincipalName, msDS-AllowedToDelegateTo, userAccountControl -ResultSetSize 100` ‚Äî SPNs, delegation flags, UAC
  - `Get-ADComputer -Filter * -Properties ServicePrincipalName, TrustedForDelegation` ‚Äî Computer SPNs and delegation
  - `Get-ADGroup -LDAPFilter "(msDS-AllowedToActOnBehalfOfOtherIdentity=*)" -Properties *` ‚Äî RBCD principals
  - `Get-ADObject -LDAPFilter "(userAccountControl:1.2.840.113556.1.4.803:=4194304)" -SearchBase (Get-ADDomain).DistinguishedName -Properties *` ‚Äî DONT_REQ_PREAUTH
- **Native Windows**:
  - `klist` ‚Äî View/verify cached tickets
  - `setspn -Q */*` ‚Äî Broad SPN discovery (use sparingly)
  - `nltest /dsgetdc:<domain>` ‚Äî Closest DC/KDC
  - `whoami /groups` ‚Äî Ticket group context validation
  - `wevtutil qe Security /q:*[System[(EventID=4769)]] /c:1` ‚Äî TGS request sanity

## üõ°Ô∏è **DETECTION, OPSEC & CLEANUP**

### üö® Detection Indicators
- Security 4768/4769 spikes (AS/TGS), unusual SPN targets, repeated S4U
- Kerberos pre-auth failures, RC4‚Äëonly patterns, service ticket bursts

### üîí OPSEC Best Practices
> **üîí STEALTH REQUIREMENT**: **Invisi-Shell** must be loaded before any Kerberos enumeration in production environments to bypass logging and AMSI detection. This is non-negotiable for operational security.
- Prefer MS‚Äësigned cmdlets first, scope with LDAP filters and small `ResultSetSize`
- Introduce jitter; avoid `setspn -Q */*` in production
- Separate DNS vs LDAP vs ADWS latency; keep transport consistent per baseline

### üßπ Cleanup

# Files and artifacts
Remove-Item spn_*.*, *asrep*.txt -ErrorAction SilentlyContinue

# Tickets and state
